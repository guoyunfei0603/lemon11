为什么要用函数？
    1. 代码逻辑中数据有的部分是写死的，不灵活，为了应对这个现象，采用参数
       有了参数，就把权力托付给了调用的人

    2. 写代码的方式：
       先写出逻辑,然后把逻辑中觉得不灵活的部分通过参数变得灵活（重点理解）
        1) 先写出逻辑
        2) def 变成函数
        3) 想办法提高代码的复用性（参数）

    3. 参数默认值以及需要注意的问题：
        1)有默认值参数要放到位置参数的后面
    4. 传递时，你喜欢带参数还是不带参数呢？？
        不带：按照顺序来传,可读性差（大多数人都喜欢这么写）
        带：可以不按顺序来,可读性好

    5. 变参（*args  **kwargs）
        定义时, * 将参数配成元组,调用时,*将元组或列表打散成参数进行参数传递
        定义时,** 将参数配成字典,调用时,**将字典打散成参数进行参数传递

    6. 返回值 return
        什么时候用？不用？
            程序员A写的函数提高数据给程序员B,C,D调用,至于他们拿到数据后怎么使用,和我没关系了
        作者写代码是这样的习惯：
            1) 我把逻辑写出来（死数据或没有死数据）
            2) 有死数据吗？有,用参数吧，依次替换
                这些需要灵活的数据需要给默认值吗？需要,给吧
                需要优雅一点吗?需要，用变参吧
            3) 这个函数是给别人用的吗？（我只是提供数据,别人怎么用是他们决定的）是,改成返回值 return
    代码对新人来说,是磨出来的,写多了，就慢慢灵活了
    7. 完整业务--用户的增删改查（重点理解） 一般都是这样写的
    注意：当把函数变成类的时候，一定要加self！
